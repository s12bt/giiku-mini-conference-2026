# mvp-todo の localStorage データスキーマ

`localStorage` に保存するデータの構造定義。**このスキーマは原則変更しない**(後で React 化する際にユーザーのデータを引き継ぐため)。スキーマを変える必要が出たら、キーを `mvp-todo:v2` のようにバージョンを上げて、 v1 → v2 のマイグレーションを書く。

## キー

```
mvp-todo:v1
```

## 値(JSON)

```json
{
  "tasks": []
}
```

## tasks の各要素(予定)

```json
{
  "id": "uuid",
  "title": "タスクのタイトル",
  "done": false
}
```

MVP 版なので、フィールドは最小限に絞る。「これがないと困る」 ものだけを足し、「あったらいいな」 で増やさないこと。
