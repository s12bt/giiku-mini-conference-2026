# super-todo の localStorage データスキーマ

`localStorage` に保存するデータの構造定義。**このスキーマは原則変更しない**(後で React 化する際にユーザーのデータを引き継ぐため)。スキーマを変える必要が出たら、キーを `super-todo:v2` のようにバージョンを上げて、 v1 → v2 のマイグレーションを書く。

## キー

```
super-todo:v1
```

## 値(JSON)

```json
{
  "tasks": []
}
```

## tasks の各要素(予定)

```json
{
  "id": "uuid",
  "title": "タスクのタイトル",
  "done": false,
  "createdAt": "ISO8601"
}
```

機能から考えて作る版なので、後でフィールドが追加されていく可能性が高い。追加する時は **既存フィールドを消さない** こと。
